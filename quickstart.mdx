---
title: "Quickstart Guide: Build Your First RoseLine Pipeline"
description: "Learn how to build your first RoseLine pipeline: define a data bag, create a step, run the pipeline, and handle results in under 5 minutes."
---

This guide walks you through building a complete RoseLine pipeline from scratch. You will define a data bag, write two processing steps, wire them into a pipeline, and handle both success and error outcomes. The examples come directly from the RoseLine test project.

<Steps>
  <Step title="Define a bag">
    A bag holds the data that flows through your pipeline. Declare a `record` that inherits `BagBase`:

    ```csharp
    using RoseLine;

    public sealed record ExampleBag : BagBase
    {
        public List<string> Users { get; set; } = [];
    }
    ```

    Bags must be `record` types. Records support the C# `with` expression, which lets each step produce a new copy of the bag with only the changed fields updated — leaving the original untouched.

    <Note>
      Using the `with` expression (`input with { Users = newList }`) creates a shallow copy of the record with the specified properties replaced. This is the recommended way to produce an updated bag inside a step.
    </Note>
  </Step>

  <Step title="Define a custom error (optional)">
    By default, RoseLine uses `StepError` for pipeline errors. You can extend it to add domain-specific fields:

    ```csharp
    using RoseLine;

    public class CustomError : StepError
    {
        public string Code { get; set; }
        public int Severity { get; set; }
    }
    ```

    `StepError` already provides `Message`, `Step` (the name of the step that failed), and `IsUnhandledError` (set automatically for uncaught exceptions). Your custom class adds any additional fields you need.

    If you skip this step, use `StepBase<TBag>` and `Pipeline<TBag>` instead of the two-type-parameter variants.
  </Step>

  <Step title="Implement your steps">
    Each step inherits `StepBase<TBag, TError>` and overrides `ProcessAsync`. Use `Next()` to advance, `Stop()` to halt cleanly, or `ThrowStepError()` to signal a handled failure.

    **Step 1 — fetch users:**

    ```csharp
    using RoseLine;

    public class GetAllUserStep : StepBase<ExampleBag, CustomError>
    {
        protected override async Task<ExampleBag> ProcessAsync(ExampleBag input)
        {
            var users = GetFakeUsers();

            if (!users.Any())
            {
                return ThrowStepError(new CustomError
                {
                    Message = "Users is not found!",
                    Code = "A1",
                    Severity = 1
                });
            }

            var output = input with
            {
                Users = users
            };

            return Next(output);
        }

        private List<string> GetFakeUsers()
        {
            return new List<string> { "Çağlar", "Osman", "Ismail" };
        }
    }
    ```

    **Step 2 — remove the first user:**

    ```csharp
    using RoseLine;

    public class DeleteFirstUserStep : StepBase<ExampleBag, CustomError>
    {
        protected override async Task<ExampleBag> ProcessAsync(ExampleBag input)
        {
            input.Users.RemoveAt(0);

            var output = input with
            {
                Users = input.Users
            };

            return Next(output);
        }
    }
    ```

    The `with` expression in each step produces a new `ExampleBag` instance, preserving the immutability guarantee of record types.
  </Step>

  <Step title="Create the pipeline and add steps">
    Instantiate a `Pipeline<TBag, TError>` with your initial bag, then add your steps in execution order:

    ```csharp
    using RoseLine;

    var bag = new ExampleBag
    {
        Users = []
    };

    var pipeline = new Pipeline<ExampleBag, CustomError>(bag);

    pipeline.AddStep(new GetAllUserStep());
    pipeline.AddStep(new DeleteFirstUserStep());
    ```

    `AddStep` returns the pipeline instance, so you can also chain calls:

    ```csharp
    pipeline
        .AddStep(new GetAllUserStep())
        .AddStep(new DeleteFirstUserStep());
    ```
  </Step>

  <Step title="Execute and handle the result">
    Call `ExecuteAsync()` and inspect the `ExecutionResult` to branch on success or failure:

    ```csharp
    var result = await pipeline.ExecuteAsync();

    if (result.IsSuccess)
    {
        // Access the final bag state
        Console.WriteLine(string.Join(",", result.Bag.Users));
        // Output: Osman,Ismail
    }
    else
    {
        // Access the typed error with step name and custom fields
        Console.WriteLine(
            $"Step: {result.Error.Step}, " +
            $"IsUnhandledError: {result.Error.IsUnhandledError}, " +
            $"Code: {result.Error.Code}");
    }
    ```

    `ExecutionResult` exposes three properties:

    | Property | Type | Description |
    |---|---|---|
    | `IsSuccess` | `bool` | `true` if all steps completed without error |
    | `Bag` | `TBag` | The final state of the data bag |
    | `Error` | `TError?` | The error from the failing step, or `null` on success |

    When a step throws an unhandled exception, RoseLine catches it automatically, sets `IsUnhandledError` to `true`, records the step name, and populates `Message` with the exception details.
  </Step>
</Steps>
