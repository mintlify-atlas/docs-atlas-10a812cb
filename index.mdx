RoseLine is a .NET library that lets you build sequential processing pipelines where each step can transform data, handle errors gracefully, and control whether execution continues or stops. It targets **net8.0** and is available on NuGet as `RoseLine`.

## Why RoseLine?

Many applications need to process data or execute tasks in a defined sequence. Managing errors, passing data between steps, and stopping execution on certain conditions can become complex fast. RoseLine gives you a structured, type-safe way to compose these workflows.

<CardGroup cols={2}>
  <Card title="Immutable data flow" icon="shield">
    Record-based `BagBase` prevents accidental mutation between steps
  </Card>
  <Card title="Structured error handling" icon="triangle-exclamation">
    `ThrowStepError` captures which step failed, the error message, and whether it was unhandled
  </Card>
  <Card title="Flow control" icon="code-branch">
    Call `Next()` to continue or `Stop()` to halt the pipeline — no exceptions needed
  </Card>
  <Card title="Async-first" icon="bolt">
    Every step uses `Task`-based `ProcessAsync` for modern async workflows
  </Card>
</CardGroup>

## Get started in 3 steps

<Steps>
  <Step title="Install the NuGet package">
    ```bash
    dotnet add package RoseLine
    ```
  </Step>
  <Step title="Define a bag and a step">
    ```csharp
    public record MyBag : BagBase { public string Value { get; set; } }

    public class MyStep : StepBase<MyBag>
    {
        protected override async Task<MyBag> ProcessAsync(MyBag input)
        {
            var output = input with { Value = "processed" };
            return Next(output);
        }
    }
    ```
  </Step>
  <Step title="Build and execute the pipeline">
    ```csharp
    var pipeline = new Pipeline<MyBag>(new MyBag { Value = "initial" });
    pipeline.AddStep(new MyStep());

    var result = await pipeline.ExecuteAsync();
    Console.WriteLine(result.Bag.Value); // processed
    ```
  </Step>
</Steps>
